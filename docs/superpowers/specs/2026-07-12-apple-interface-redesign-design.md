# 观象 Apple 风格界面重塑设计

## 目标

在不改变起卦、分析、历史记录和报告逻辑的前提下，把现有偏仿古的研究工具界面重塑为现代 Apple 产品式体验：轻盈、精确、层级清楚、触控友好，并在桌面与移动端保持一致品质。

## 设计方向

采用“Apple 现代骨架＋极少量东方识别”。界面以真实白色、`#F5F5F7` 浅灰背景和 `#1D1D1F` 主文字为基础，主操作使用 `#0071E3` Apple 蓝。东方识别只保留在“观象”品牌、卦象线条与内容本身，不再依赖米色纸张、朱砂色块、金色装饰或大面积宋体。

比较过的方向：

1. 纯白 Apple 工具风：最清爽，但品牌辨识度不足。
2. 深色 Pro 工具风：结果页有冲击力，但问事流程长期使用偏压迫。
3. Apple 现代骨架＋东方内容识别：兼顾高级感、可读性和产品独特性，作为最终方向。

## 信息架构

信息架构保持不变：顶部品牌与记录入口、三步起卦、问事表单、起卦方式、数源复核、完整结果、证据链和历史抽屉。布局从“仿古研究桌”调整为“Apple 产品配置器”：

- 顶部使用半透明玻璃导航，品牌和操作保持克制。
- 起卦页使用居中的宽内容区，步骤导航变成紧凑的胶囊式进度轨道。
- 主面板使用一块主要浮层，不把每个字段包装成独立卡片。
- 结果页使用高对比总结区、开放式内容区和轻量分隔线，减少深色大块与边框堆叠。
- 历史抽屉采用系统侧边面板语言，突出记录标题、收藏和比较动作。

## 视觉系统

- 字体：`-apple-system`、`BlinkMacSystemFont`、`SF Pro Display/Text`、`PingFang SC`；所有标题与正文统一使用现代无衬线。
- 颜色：背景 `#F5F5F7`，表面 `rgba(255,255,255,.86)`，主文字 `#1D1D1F`，次文字 `#6E6E73`，边线 `rgba(0,0,0,.08)`，主操作 `#0071E3`，成功 `#248A3D`，警示 `#FF9F0A`，风险 `#FF3B30`。
- 圆角：主面板 28px，次面板 20px，控件 14px，胶囊按钮 999px。
- 阴影：只用低不透明度、大模糊半径的环境阴影；悬浮不产生明显跳动。
- 控件：输入高度不少于 52px，按钮高度不少于 48px，蓝色主按钮为胶囊形，聚焦环使用半透明 Apple 蓝。
- 动效：页面进入、面板切换和悬停采用 180–320ms 缓动，并尊重 `prefers-reduced-motion`。

## 关键页面

### 问事页

标题更大、更简洁，问题输入作为主要焦点。三个上下文字段在桌面并排、移动端纵向排列。说明文字降低对比度。步骤轨道在桌面位于主面板左侧或顶部，在窄屏保持横向且不挤压文字。

### 起卦方式与复核页

六种方式使用轻量分段卡片，选中态以浅蓝底、蓝色描边和细微高光表达。复核页的本互变卦以三列开放式结构呈现，避免仿古边框。

### 结果页

总结区使用接近黑色的高对比背景与柔和蓝色光晕，分数环保持清晰。后续区块使用白色开放面板与细分隔线；本互变关系、行动建议和证据链保留完整内容，但通过字号、间距和对比度建立层级。

### 移动端

390×844 为主要验收尺寸。导航压缩为图标＋必要文字，所有按钮满宽或可自然换行，结果三列改为纵向，证据链不抢占首屏，不产生横向滚动。

## 技术边界

- 不修改领域计算、卦象数据、分析规则和历史序列化格式。
- 在 `src/apple-theme.css` 中隔离新视觉层，并在 `src/main.tsx` 中最后引入。
- `App` 仅增加明确的主题标识，现有组件结构只做满足布局和可访问性所需的最小调整。
- 使用现有 Lucide 图标，不增加新的运行时依赖。

## 测试与验收

- 自动化测试先约束 Apple 主题标识和现有核心流程不回归。
- 运行 `npm test`、`npm run typecheck`、`npm run build`。
- 使用内置浏览器在桌面和 390×844 下检查首屏、完整起卦流程、结果页、历史抽屉、控制台和横向溢出。
- 将浏览器截图与视觉概念逐项比较：布局、字体、颜色、圆角、控件、间距、结果页层级和移动端折叠。
