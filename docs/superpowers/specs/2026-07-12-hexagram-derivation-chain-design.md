# 上下卦象完整推导链设计

## 目标

把当前仅有三条数源求和的“数源公式”，升级为可以逐步人工复算的卦象推导链。在第 3 步校验页和最终结果页使用同一套数据与组件，完整说明数字怎样转为上下卦、本卦怎样生成互卦与变卦、动爻怎样确定体用，同时保持现有起卦结果和历史记录兼容。

## 方案比较

1. 直接扩写 `casting.calculation` 文本：改动最少，但结构不可测试，桌面与移动端难以形成清晰层级，后续容易出现文案与真实计算不一致。
2. 校验页和结果页分别增加说明：视觉可分别优化，但会形成两套推导逻辑，维护时容易前后矛盾。
3. 建立纯函数推导模型与共享组件：从 `CastingResult` 生成结构化步骤，由两个页面共同渲染；计算、文案和测试都有单一来源。采用此方案。

## 推导内容

完整推导分为六段，所有结论必须由当前卦例的真实数据生成。

### 1. 数源组成

- 按起卦方式列出上卦、下卦、动爻各自取数来源。
- 时间起卦显示年支序数、农历月、农历日、时支序数及逐项求和。
- 报数、文字、物数、物象方位分别保留各自的取数口径。
- 自定义排盘明确标记为“直接指定”，不伪造除法取余过程。

### 2. 除法取余与先天数映射

- 上卦数与下卦数分别除以 8，显示被除数、除数、商、原始余数和归一化结果。
- 原始余数为 0 时明确显示“余 0 按 8 取数”，避免用户误认为没有对应卦。
- 显示先天八卦数表：`1乾、2兑、3离、4震、5巽、6坎、7艮、8坤`，并高亮本次使用的两个数。
- 每个所得经卦显示卦名、符号、三爻阴阳、自然象、五行、方位、家庭角色与关键词。

### 3. 上下卦合成本卦

- 明确上卦放在六爻的第 4–6 爻，下卦放在第 1–3 爻；爻序始终自下向上编号。
- 同时展示上下卦三爻序列与合成后的六爻序列。
- 显示所得六十四卦名称，并说明上卦偏向外部、上层或显现条件，下卦偏向内部、基础或起始条件；该位置含义只作传统结构参考，不替代具体问事语境。

### 4. 互卦取爻

- 显示下互取本卦第 2、3、4 爻，上互取第 3、4、5 爻。
- 展示两组三爻阴阳序列、各自映射的经卦，以及重新组合后的互卦名称。
- 解释互卦用于观察结构内部的牵引与演化机制，不机械等同于固定日期上的“过程”。

### 5. 动爻与变卦

- 动爻总数除以 6，显示商、余数及余 0 按第 6 爻处理的规则。
- 标明动爻位于上卦或下卦、原爻为阴或阳、翻转后为何爻。
- 在六爻图中并排显示“本卦 → 单爻翻转 → 变卦”，只高亮发生变化的一爻。
- 显示变化发生在哪个经卦、原经卦如何变为新经卦，以及最终变卦名称。

### 6. 体用定位与一致性校验

- 动爻所在经卦为用，另一静卦为体；明确展示本次体卦、用卦及其五行。
- 显示体用生克关系，但月令旺衰与事项语境仍交由现有分析模块解释，推导链不重复生成吉凶结论。
- 校验上卦数、下卦数、动爻数、互卦取爻、变卦翻爻和卦名重建结果；全部通过时显示“推导一致”。
- 任一步数据缺失或不一致时显示具体异常项，不以绿色完成态掩盖问题。

## 数据结构

新增纯函数 `buildCastingDerivation(casting)`，返回只读的结构化推导对象。它负责：

- 标准化三项取余校验；
- 输出先天八卦映射信息；
- 输出本卦六爻、互卦取爻与变爻前后序列；
- 确定动爻所在经卦及体用；
- 汇总逐项一致性校验。

推导对象在渲染时计算，不写入 `CastingResult`，因此不会改变历史记录存储格式。旧记录如果缺少 `calculationChecks`，推导函数根据现有上下卦数与动爻重建必要信息；无法可靠还原的原始数源明确显示“旧记录未保存该项”，不臆造数据。

## 组件与界面

新增共享 `CastingDerivation` 组件：

- `compact` 模式用于第 3 步校验页，默认完整展示六段，但压缩解释文字。
- `full` 模式用于最终结果页，替换当前只有数源公式的左栏，默认展开且允许用户折叠。
- 顶部放置六段步骤轨道；每段包含公式、结果、卦象图与一段解释，避免大段连续文字。
- 桌面端使用双列或三列步骤卡；小于 820px 时改为单列，不出现横向滚动。
- 阴阳爻使用现有 `HexagramGlyph` 视觉语言；取爻与变爻增加编号和高亮，不新增图片资源或运行时依赖。

结果页原“解释边界”保留，并与推导链分栏：推导链回答“怎么算出来”，解释边界回答“可以怎样理解、不能怎样机械理解”。复制报告同步增加简明的取余、上下卦、互卦、变卦和体用推导摘要。

## 错误处理

- 自定义排盘只显示直接指定路径、六爻组合、互卦、变卦和体用，不显示虚假的商与余数。
- 余数归一化、三爻识别、六十四卦名称或变爻重建不一致时，将对应步骤标记为异常并给出实际值与期望值。
- 组件只消费推导对象，不在 JSX 中重新计算关键结果，防止不同页面产生不同答案。

## 测试与验收

- 领域测试覆盖余 0 按 8、余 0 按 6、普通余数、上下卦合成、互卦 2–4/3–5 取爻、上下卦动爻、阴阳翻转、体用定位和自定义排盘。
- 组件测试确认校验页与结果页展示相同的关键推导数据，并确认旧记录缺少原始数源时使用明确的缺失提示。
- 报告测试确认复制文本包含取余、先天卦映射、互卦、变卦和体用摘要。
- 运行 `npm test`、`npm run typecheck`、`npm run build`。
- 使用浏览器检查桌面与 390×844：步骤完整、默认展开、可折叠、无横向溢出，卦名、爻位和体用结果与自动测试一致。
